# 跨设备运行计划书

## 背景

当前程序在**同一台电脑**上运行（`127.0.0.1` 回环地址）。若让客户端和服务器分别在**不同设备**上运行，评估如下。

---

## 工作难度：极低

核心代码**无需改动**，只需修改配置。

---

## 代码分析

### 服务器端（无需改动）

```c
// src/server.c 第 182 行
addr.sin_addr.S_un.S_addr = htonl(INADDR_ANY);
```
`INADDR_ANY` 表示接受任意网卡的连接，天然支持跨设备访问。

服务器绑定 `0.0.0.0:6000`，监听所有网络接口。

### 客户端端（改动 1 行）

```c
// src/client.c 第 11 行
#define DEFAULT_IP   "127.0.0.1"   // ← 改为服务器实际 IP
```

目前写死了 `127.0.0.1`（本机回环），需要改成服务器所在电脑的局域网 IP。

---

## 实施步骤

### Step 1：确认网络环境

两台电脑连接到**同一个局域网**（同一 WiFi 或网线直连）。

### Step 2：查看服务器 IP

在服务器电脑上打开 CMD，运行：

```cmd
ipconfig
```

找到 IPv4 地址，格式类似 `192.168.x.x`，例如 `192.168.1.100`。

### Step 3：修改客户端 IP

将 `src/client.c` 第 11 行：

```c
#define DEFAULT_IP   "192.168.1.100"   // 改成服务器的实际 IP
```

### Step 4：放行防火墙

在服务器电脑上：

1. Win + R → `wf.msc` 打开 Windows 防火墙高级设置
2. 入站规则 → 新建规则 → 端口 → 6000 → 允许连接
3. 完成

或者命令行快速放行：

```cmd
netsh advfirewall firewall add rule name="TCP_6000" dir=in action=allow protocol=tcp localport=6000
```

### Step 5：重新编译

编译命令不变，重新生成 `client.exe`：

```cmd
cl /EHsc /Fe:bin\client.exe src\client.c ws2_32.lib
```

### Step 6：运行

- 服务器电脑：运行 `bin/server.exe`
- 客户端电脑：运行 `bin\client.exe`

连接成功后，服务器端会显示客户端的**真实局域网 IP**（如 `192.168.1.101:xxxxx`），不再是 `127.0.0.1`。

---

## 对比总结

| 项目 | 当前（单设备） | 跨设备运行 |
|------|-------------|----------|
| 服务器 IP | `127.0.0.1`（本机） | 局域网 IP（如 `192.168.1.100`） |
| 客户端 IP | `127.0.0.1`（本机） | 局域网 IP（如 `192.168.1.101`） |
| 网络要求 | 无 | 同一局域网 |
| 代码改动 | 无 | 仅改客户端 1 行 IP |
| 防火墙 | 无需配置 | 需放行端口 6000 |
| 服务器端代码 | 不用改 | 不用改 |

---

## 注意事项

- 服务器 IP 必须是**局域网 IP**，不是公网 IP
- 如果在同一 WiFi 下但不在同一局域网（比如 guest 网络隔离），可能无法连接
- 跨公网连接需要做端口映射，较复杂，不在本次讨论范围
