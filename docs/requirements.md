# 实验需求文档

来源：《计算机网络》课程综合性实验指导单（撰写人：陈振国，2026年3月）

适用专业：物联网工程

---

## 一、实验目的

1. 熟悉和掌握网络编程的基本方法和步骤
2. 进一步理解 client/server 交互模式
3. 加深对网络协议概念的理解以及协议的设计和实现方法

---

## 二、设备与环境

- 微型计算机
- Windows 系列操作系统
- 开发语言：C（或 C++、JAVA）
- 开发环境不限（推荐 VC++ / MinGW）

---

## 三、实验内容

使用 C（或 C++、JAVA）语言编写网络通信模拟程序，由 **client** 和 **server** 两部分组成，client 和 server 之间的通信基于 **TCP 或者 UDP 协议**，要求能够实现客户和服务器端的交互对话。

### 原始需求清单

> 以下为实验指导单原文，逐条编号：

| 序号 | 原始需求（指导单原文） |
|------|----------------------|
| 1 | 服务器和客户端有提示 |
| 2 | 服务器启动后处于监听状态等待客户机连接，当有客户连接时给出提示，并显示客户 IP 地址，并给客户端发送问候消息 |
| 3 | 客户机端要求启动后主动连接服务器，也可以通过菜单控制连接的时机，连接成功后给出提示 |
| 4 | 在通信连接建立之后，客户和服务器双方可以进行双向消息传递，直至一方发送 `quit` 消息，断开连接 |
| 5 | 要求界面尽量友好，功能尽量完善，可考虑增加功能菜单，提高系统的交互性 |
| 6 | 对功能进行适当分解，用若干函数完成系统 |

### 需求与实现对照表

| 编号 | 原始需求 | 程序实现 | 状态 |
|------|---------|---------|------|
| R1 | 服务器和客户端有提示 | `server.c` main() 打印启动横幅；`client.c` show_menu() 打印菜单 | ✅ |
| R2 | 服务器监听，显示IP，发送问候 | `accept()` + `inet_ntoa()` 显示IP + `send_greeting()` 发送欢迎消息 | ✅ |
| R3 | 客户端主动连接，菜单控制时机 | `connect_server()` + `show_menu()` 循环选择 1-4 | ✅ |
| R4 | 双向消息传递，quit 断开 | 多线程 `handle_client()` / `recv_thread()` + `strcmp("quit")` 触发断开 | ✅ |
| R5 | 界面友好，功能分函数 | 模块化 `static` 函数结构，每个函数单一职责 | ✅ |
| R6 | 基于 TCP 协议 | `socket(AF_INET, SOCK_STREAM, 0)` | ✅ |

1. 用 VC++ 开发工具编程实现上述实验内容中的结构定义和算法

## 六、参考代码说明

实验指导单附录提供了 C 语言 Winsock 参考代码，涵盖：

| 函数 | 作用 |
|------|------|
| WSAStartup / WSACleanup | Winsock 初始化和清理 |
| socket | 创建 TCP 流式套接字或 UDP 数据报套接字 |
| bind | 服务器绑定本地地址和端口 |
| listen | 服务器开始监听连接请求 |
| accept | 服务器接受客户端连接 |
| connect | 客户端请求连接服务器 |
| send / recv | 发送和接收数据 |
| closesocket | 关闭套接字 |

端口：6000，服务器 IP：127.0.0.1（本地回环）
