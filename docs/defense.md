## 一、基础概念类

### Q1：计算机网络按覆盖范围分为哪几类？请举例。

| 类型 | 覆盖范围 | 举例 |
|------|---------|------|
| PAN | 个人区域 | 蓝牙 |
| LAN | 局域网 | 办公室/校园网 |
| MAN | 城域网 | 城市WiFi |
| WAN | 广域网 | Internet |

**本程序运行在本地 localhost (127.0.0.1)，属于 LAN 范畴。**

---

### Q2：计算机网络有哪些性能指标？

- **带宽 (Bandwidth)**：单位时间内能传输的最大数据量，单位 bps
- **时延 (Delay)**：数据从发送端到接收端的时间，包括发送时延、传播时延、处理时延、排队时延
- **吞吐量 (Throughput)**：实际传输的数据量
- **往返时延 RTT**：数据包往返一次的时间

**TCP 传输相比 UDP 更可靠，但时延略高。**

---

### Q3：什么是网络体系结构？为什么要分层？

采用分层结构，将复杂问题分解为多个相对独立的子问题：

| 体系结构 | 特点 |
|---------|------|
| OSI 七层 | 理论标准，概念完整 |
| TCP/IP 四层 | 事实标准，广泛应用 |

本程序涉及 **运输层**（TCP/UDP 提供端到端服务）和 **应用层**（socket 编程接口）。

## 二、物理层与数据链路层

### Q4：信道复用技术有哪些？

| 技术 | 原理 | 应用场景 |
|------|------|---------|
| FDM | 频分多路复用，按频率划分 | 传统电话线 |
| TDM | 时分多路复用，按时间片划分 | 数字传输 |
| WDM | 波分多路复用，按波长划分 | 光纤通信 |
| CDM | 码分多路复用，靠编码区分 | 移动通信 |

本程序未直接涉及，但在网络中普遍使用。

---

### Q5：什么是 CSMA/CD？作用是什么？

**Carrier Sense Multiple Access with Collision Detection**
载波监听多路访问/冲突检测

- 作用：解决总线型局域网中多台设备同时发送数据的冲突问题
- 原理：先监听信道，空闲则发送，发送时检测冲突，冲突后等待随机时间重发
- **本程序使用的是 TCP，不需要 CSMA/CD**，TCP 的冲突由运输层处理

---

### Q6：PPP 协议和以太网帧的区别？

- **PPP**：点对点协议，用于拨号或专线连接，面向字节，3 层协议字段，无帧序号
- **以太网帧**：广播型网络，MAC 地址，6 字节源/目的地址，支持 CSMA/CD

本程序 socket 编程工作在 **运输层**，不直接操作链路层帧。

## 三、网络层

### Q7：IP 地址和 MAC 地址的区别？

| | IP 地址 | MAC 地址 |
|---|---------|---------|
| 层次 | 网络层（逻辑地址） | 数据链路层（物理地址） |
| 分配 | 可配置，会变 | 固化在网卡，出厂固定 |
| 作用 | 全网寻址路由 | 局域网内寻址 |
| 长度 | IPv4 32位，IPv6 128位 | 48位 |

**本程序中 `inet_ntoa()` 把网络层 IP 地址转换为点分十进制字符串显示。**

---

### Q8：为什么要划分子网？

将一个大的 IP 网络划分为多个小的子网，提高 IP 地址利用率，增强网络管理和安全性。

**本程序绑定 `INADDR_ANY`，表示接受本机任意网卡的连接。**

---

### Q9：ARP 协议的作用是什么？

**Address Resolution Protocol**，将 IP 地址解析为 MAC 地址。
当主机知道目标 IP 但不知道 MAC 地址时，广播 ARP 请求，收到应答后缓存。

---

### Q10：RIP 和 OSPF 的区别？

| | RIP | OSPF |
|---|-----|------|
| 类型 | 距离矢量协议 | 链路状态协议 |
| 跳数上限 | 15跳 | 无限制 |
| 更新方式 | 定期广播整个路由表 | 状态变化时更新 |
| 适用 | 小型网络 | 大型/中型网络 |

---

## 四、运输层（重点）

### Q11：TCP 和 UDP 的区别？

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接性 | 面向连接（三次握手） | 无连接 |
| 可靠性 | 可靠（确认、重传、流控） | 不可靠 |
| 顺序 | 有序 | 不保证顺序 |
| 速度 | 较慢 | 较快 |
| 头部 | 20-60字节 | 8字节 |
| 适用场景 | 文件传输、网页、邮件 | 视频、语音、DNS |

**本程序使用 TCP**（`SOCK_STREAM`），适合交互式对话。

---

### Q12：TCP 三次握手过程是什么？

```
客户端  ──────────────────────────────>  服务器
       SYN=1, seq=x (第一次握手)
客户端  <──────────────────────────────  服务器
       SYN=1, ACK=1, seq=y, ack=x+1 (第二次握手)
客户端  ──────────────────────────────>  服务器
       ACK=1, seq=x+1, ack=y+1 (第三次握手)
```

- 第一次：客户端发送 SYN，请求建立连接
- 第二次：服务器返回 SYN+ACK
- 第三次：客户端发送 ACK，连接建立成功

**本程序 `connect()` 触发三次握手，`accept()` 返回后连接已建立。**

---

### Q13：TCP 四次挥手过程是什么？

```
客户端  ──────────────────────────────>  服务器
       FIN=1, seq=u (第一次挥手)
客户端  <──────────────────────────────  服务器
       ACK=1, ack=u+1 (第二次挥手)
       ... (服务器可能还有数据要发) ...
客户端  <──────────────────────────────  服务器
       FIN=1, seq=w (第三次挥手)
客户端  ──────────────────────────────>  服务器
       ACK=1, ack=w+1 (第四次挥手)
       ... 等待 2MSL 后关闭 ...
```

**本程序中 `closesocket()` 触发四次挥手释放连接。**

---

### Q14：TCP 流量控制是怎么实现的？

通过 **滑动窗口** 机制：
- 接收方在 TCP 头部字段中通告自己的接收窗口大小
- 发送方根据接收方窗口大小调整发送速率
- 窗口为 0 时发送方停止发送，直到收到非零窗口通知

---

### Q15：TCP 拥塞控制有哪些机制？

| 机制 | 说明 |
|------|------|
| 慢启动 | 拥塞窗口从小开始指数增长 |
| 拥塞避免 | 窗口达到阈值后线性增长 |
| 快重传 | 收到三个重复 ACK 立即重传 |
| 快恢复 | 拥塞后不重新慢启动 |

**本程序使用 `send()`/`recv()`，拥塞控制由操作系统自动处理。**

---

### Q16：UDP 有三次握手吗？为什么？

UDP **没有**三次握手，因为 UDP 是无连接的，不需要建立连接直接发送数据。因此 UDP 比 TCP 开销更小、速度更快，但不可靠。

## 五、Socket 编程（本程序核心）

### Q17：Winsock 编程的步骤是什么？

**服务器**：
1. `WSAStartup()` 初始化
2. `socket()` 创建套接字
3. `bind()` 绑定地址和端口
4. `listen()` 开始监听
5. `accept()` 接受连接
6. `send()`/`recv()` 收发数据
7. `closesocket()` 关闭连接
8. `WSACleanup()` 清理

**客户端**：
1. `WSAStartup()`
2. `socket()`
3. `connect()` 连接服务器
4. `send()`/`recv()`
5. `closesocket()`
6. `WSACleanup()`

---

### Q18：`socket(AF_INET, SOCK_STREAM, 0)` 各参数含义？

- `AF_INET`：IPv4 地址族
- `SOCK_STREAM`：字节流套接字 = **TCP**
- `protocol = 0`：自动选择，默认 TCP

如果用 `SOCK_DGRAM` 就是 UDP。

---

### Q19：`bind()` 时 `htonl(INADDR_ANY)` 是什么意思？

`INADDR_ANY` 表示绑定本机的**所有网络接口**（有线、无线、虚拟机等）。
`htonl()` 将主机字节序转换为网络字节序（big-endian）。

对应本程序 `src/server.c` 第 182 行。

---

### Q20：`htonl`、`htons`、`ntohl`、`ntohs` 的区别？

| 函数 | 含义 | 用途 |
|------|------|------|
| `htonl` | host to network long | IP 地址的字节序转换 |
| `htons` | host to network short | 端口号的字节序转换 |
| `ntohl` | network to host long | 反向转换 IP |
| `ntohs` | network to host short | 反向转换端口 |

**为什么需要转换？** 不同机器的字节序不同（大小端），网络统一用大端序。

---

### Q21：`select()` 函数的作用是什么？

`select()` 可以同时监控多个 socket 是否可读/可写/异常。

**本程序用它实现非阻塞接收**（`src/server.c` 第 102 行，`src/client.c` 第 133 行）——不阻塞等待网络数据，每 0.1 秒检查一次。

---

### Q22：`_beginthreadex()` 和 CreateThread 的区别？

| | `_beginthreadex()` | `CreateThread()` |
|---|---|---|
| 库 | C 运行时库 | Windows API |
| fd 句柄 | 自动管理 | 需手动管理 |
| 与 C 库配合 | 安全 | 可能冲突 |

**本程序使用 `_beginthreadex()`（`src/server.c` 第 67 行）。**

---

### Q23：多线程在网络编程中的作用是什么？

- 服务器可以为每个客户端创建一个独立线程，同时处理多个连接
- 客户端可以用一个线程发送、一个线程接收，实现**全双工通信**

**本程序亮点**：服务器每 `accept()` 一个客户端就创建 `handle_client()` 线程，在线程内用 `select()` 接收 + `_kbhit()` 键盘输入，实现双向对话。

---

### Q24：端口号的作用是什么？为什么是 6000？

端口号用来区分同一台主机上的不同网络应用进程（16位，0-65535）。

- 0-1023：知名端口（HTTP 80，HTTPS 443，DNS 53）
- 1024-49151：注册端口
- 49152-65535：动态私有端口

**本程序使用 6000，属于注册端口范围，可自由使用。**

## 六、常见追问

### Q25：为什么选择 TCP 而不是 UDP？

- TCP 提供可靠传输，保证数据有序、无丢失、无重复
- 交互式对话需要可靠的消息传递
- 演示场景下数据量小，TCP 的开销可接受

**反问**：如果做实时游戏或视频通话，通常用 UDP，因为允许少量丢包，追求低延迟。

---

### Q26：本程序有没有考虑安全问题？

诚实回答：没有。生产环境需要考虑：
- 用户名/密码认证
- 数据加密（SSL/TLS）
- 防止 SQL 注入/XSS（如果涉及数据库）
- 防火墙限制端口访问

本次实验重点是**网络编程方法的掌握**，安全是后续方向。

---

### Q27：如果客户端异常退出，服务器会怎样？

本程序在 `recv()` 返回 0 或错误时，会 `closesocket()` 并退出线程，服务器继续监听其他客户端，不会崩溃。

---

### Q28：本程序能同时支持多个客户端吗？

技术上**可以**，因为每 `accept()` 一个客户端就创建一个线程。但当前实现中，一个线程结束后资源已释放，若要真正支持多客户端，需要用数组或链表管理所有 `sock_conn`。

### Q30：介绍一下你的程序的整体设计思路？

> 回答示例：
>
> 我的程序分为服务器和客户端两部分，均使用 C 语言和 Winsock2 编写，基于 TCP 协议实现网络通信。
>
> 服务器端：初始化 Winsock → 创建 socket → bind 绑定 6000 端口 → listen 监听 → accept 接受连接 → 创建线程处理客户端。在线程内，使用 select() 非阻塞检测网络数据，_kbhit() 检测键盘输入，从而实现服务器和客户端的双向同时通信。
>
> 客户端端：初始化 Winsock → 创建 socket → connect 连接服务器 → 创建接收线程 → 主线程通过菜单循环允许用户发送消息。
>
> 核心亮点是多线程实现了真正的双向对话，相比单线程版本，双方都能主动发送消息而互不阻塞。
